SEO基础

发表于 2018-03-07

内容

三大标签

概念介绍

1.源代码

源代码简单来理解就是浏览器解释网页给搜索引擎看的“语言”，搜索引擎看到的东西和我妈看到的不一样。

2.title标签

<title>搜狐-中国最大的门户网站</title>

3.keywords标签

<meta name="keywords" content="搜狐，门户网站，新媒体，网络媒体，新闻，财经，体育，娱乐..."/>

4.description标签

<meta name="Description" content="搜狐是全球最大的中文门户网站"/>

作用

1.title标签：

这是一个页面中最重要的标签之一，搜索引擎判断页面属性，最为重要的参考。

2.keywords标签

它是关键词的意思，但只是放置关键词的地方，以前这个标签很重要，现在已经被搜索引擎放弃，甚至成为判断网站是否过渡优化的依据。keywords标签建议不写任何东西。

3.description标签

它自身对SEO的影响可以忽略不计，但是会直接关系到用户体验和点击率，而这些确实会影响到SEO。

关键词

概念

SEO真正的价值是带来定向的客户，定向客户是通过搜索一些关键词来到我们网站的，这些客户要搜索的词语就是关键词。

分类

1.主关键词

我们用最高权重页面（比如首页）去优化的词我们叫做主关键词，使我们将来的网站客户客户搜索量最大、商业价值也最大的词。比如：“手机”

2.长尾关键词

更主关键词有关，但搜索量没有它大的一些词，一般用来做网页优化。比如：“诺基亚手机在北京哪里有卖？”

注意：主关键词和长尾关键词都是相对的。比如“长城汽车”这个词，在长城汽车官网就当做主关键词来优化，但是它对于汽车之家这个网站就是一个长尾关键词。

文章

文章主要由文章标题和文章内容组成。

文章标题

  1. 给人看的
  2. 给SEO搜索引擎看的，在源代码中的部分

文章内容

  1. 给人看的
  2. 给SEO搜索引擎看的，为了让给搜索引擎更容易抓取，排名更高。文章是我们优化网站的一个必要手段，在“内容为王”的基础上，对于搜索引擎来讲文章就是最终要的。

分为 

1). 原创：搜索引擎收录过的
2). 伪原创：复制粘贴原创的文章

用户体验

在上述讲文章对于搜索引擎的重要性，但可想而知，搜索引擎是看不懂文章的，所以它只能从用户行为来推断网页的质量。
举例：

  1. 用户在网页上停留的时间
  2. 点击该网页的数量
  3. 用户的收藏数量
  4. 别的网页对该网页的推荐数量

网页的质量提高，对于该网站的信任度也就提高了。

链接

概念

类如 https://www.baidu.com

反向链接

A页面上有B页面的链接，就交A给B做了一个反向链接

任何一个链接都是某一个页面对另一个页面的反向链接。任何一个超链接都是对另一个页面的信任投票。

锚文本

在文本上加上超链接，比如：百度

这就是对另一个页面最好的信任投票，直接告诉了搜索引擎这个链接的属性。

URL超链接

比如：https://www.baidu.com

url就是网站的地址

文本链接

比如： https://www.baidu.com

搜索引擎是根据(超链接周围文字)来判断该超链接是对谁的一个信任

分类

内链

是反向链接的一种重要形式（首页和内页）

http://yimi.com/      首页
http://yimi.com/seo   内页

作用：

  1. 它可以上搜索引擎更多的抓取我们的页面，以及可以让网页之间相互进行投票
  2. 内链具备相关、稳定的特点

但一个网页总共内链不易过多

外链

是反向链接的一种形式，主要指在别的网站网页上对我们网站王页进行的信任投票。

作用：

  1. 反映了该网站的受欢迎程度：在同等条件下，外链数越多，排名和稳定性越好。同等条件下，越是相关的页面越有价值。
  2. 外链是反向链接的重要组成部分：是搜索引擎对我们网站权威性的评价。

内容为王，外链为皇

应用

网站地图

搜索引擎看网站就是看连接，所以由我们网站上所有网页组成的“链接地图”就是我们的网站地图

作用：

增加搜索引擎查阅我们网站的效率，可以显著提升我们网站在搜索引擎面前的曝光率，增加搜索引擎对我们网站的到访次数。类似有引导的作用

位置：

举例：百度>淘宝>最底部，网站地图

死链接

打不开的链接，比如之前一个可以打开的页面被删除了

html

html基础

概念

HTML（Hypertext Markup Language）是用来描述网页的一种标记性语言，是一种规范，一种标准，它通过标记符号还显示网页中的各个部分。

  1. HTML指超文本标记语言
  2. HTML不是一种编程语言，而是一直标记语言
  3. 利用一套标记符号（标签），来描述网页
  4. 组成文件的后缀为.html或.htm

语法规则

  1. <元素名>内容</元素名>
  2. <元素名/>

网页结构

<!DOCTYPE>
<html>
<head>
<title>...</title>
</head>
<body>
<p>...</p>
</boody>
</html>

静态化

静态网页和动态网页

  1. 静态网页：服务器上存在的页面

  2. 动态网页：通过执行asp、php、jsp、.net等程序生成客户端页面代码的网页 

    关键词：架构与内容分离、磨具、转换生成

过程：数据库>动态网页>静态网页

静态网页和动态网页的优劣势：

静态网页

优势： 访问速度快，可以跨平台和服务器

劣势： 占用空间资源多，页面多，难以管理，修改不方便

动态网页

优势： 通过用户请求返回数据库页面，网站易于维护

劣势： 页面不真实存在，对搜索引擎不友好

静态化作用

  1. 有利于搜索引擎抓取（动态页面静态化）
  2. 提高程序性能（静态页面打开速度快）

静态化处理方式

  1. 插件静态化
  2. 代码静态化
  3. HttpWebRequest返回页面重定向法

Nofollow

概念：nofollow是一个html标签的属性。这个标签的意义是告诉搜索引擎“不要追踪此网页上的链接”或“不要追踪此特定的链接”

<a rel="nofollow" href="/index.html"></a>

作用：是站长管理和搜索引擎之间对待一个页面中链接或特定链接的方式。

解读：比如A页面中有一个链接指向B页面，如果我给这个链接加上nofolloe属性，那么A页面的这个链接就不属于B页面的反向链接，并且也不会分散A页面的权重。

权重传递概念–百度知道
