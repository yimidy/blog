发表于 2018-03-12

API

一、定义

可以理解为一些应用程序的预留接口，或者更通俗的说是一些预先编译好的函数

比如你写了一个库，里面有很多函数，如果别人要使用你这个库，但是并不知道每个函数内部是怎么实现的。使用的人需要看你的文档或者注释才知道这个函数的入口参数和返回值或者这个函数是用来做什么的。对于用户来说 ，你的这些函数就是API。

二、举例说明

本人也是在自己理解基础上做的一些分析，如有不对，欢迎来踩

框架提供微信原生API

它可以连接微信提供的一些功能，如读取用户信息、本地存储、支付功能等等。

1.wx.on开头的API是监听某个事件发生的API接口，它会接受一个callback函数作为参数。该事件触发时就会调用这个函数。

如监听WebSocket连接是否打开事件 

//先创建一个 WebSocket的连接
wx.connectSocket({
  url: 'test.php'
})
wx.onSocketOpen(function(res) {
  console.log('WebSocket连接已打开！')
})
wx.onSocketError(function(res){
 console.log('WebSocket连接打开失败，请检查！')
})

2.如果没有特殊的事件发生，一般API接口都会接受一个object作为参数。

object中一般指定success、fail、complete等函数来接收调用结果。

如以wx开头的API设置本地数据的缓存

wx.setStorage({
 key：'user',  //本地中缓存指定key
 data:'yi.du'
})

如以wx开头的API获取本地数据的缓存

wx.getStorage({
 key：'user',
 success:function(res){   //回调函数，     res={data:key对应的内容}
   console.log(res.data);  //yi.du
 }
})

三、总结

API一般适用于操作系统中，或大型的开发项目。

API是一组命令、函数、协议和对象，程序员可以用它们来开发软件或与外部系统进行交互。它为开发人员提供了执行通用操作的标准命令，这样他们就不必从头开始编写代码了。

比如如果该某程序员想通过自己的代码与你的代码进行交互，那么你将需要构建一个API。这方便于其他开发人员，甚至你的团队使用你的代码。
