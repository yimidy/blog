跨域请求

含义

  1. 跨域是一个域下的文档或脚本试图去访问另一个域下的资源。
  2. 它是由浏览器的同源政策施加的安全限制

跨域的解决方案

  1. 通过jsonp跨域 ———————–只能get请求，单向
  2. document.domain+iframe跨域 —-父子域，子域双向传值
  3. location.hash+iframe跨域———-同域不同页面，不同域之间单向传值
  4. window.name+iframe跨域———-不同域之间双向传值
  5. postMessage跨域——————-
  6. 跨域资源共享(CORS)—————
  7. nginx代理跨域————————
  8. nodeis中间件代理跨域————–
  9. WebSocke协议跨域—————–

jsonp

jsonp只能用get请求

1.实现原理：类如标签可以写入html文件里，添加src属性引入不同域下的文档渲染网页。基于此原理，我们可以动态创建script标签，在添加一个带参网址实现跨区请求。

2.原生代码如下：

<script>
//创建标签
var script = document.createElement("script");
script.type = 'text/javascript';
//传参并指定成功时服务器的·回调函数
script.src = 'http://www.domain.com:8080/login?user=admin&callback=onBack';
//动态添加标签
document.head.appendChild(scipt);
//回调执行函数
function onBack(res){
  console.log(JSON.stringify(res));
}
</script>

同时后台服务器端代码如下：

<?php
$callback = $_GET['callback'];//得到回调函数
$data = {"user":"admin"};//得到的数据
echo $callback.'(".json_encode($data).")';//输出
?>

成功时服务端返回如下：

onBack({"user":"admin"})

所以通过http://www.domain.com:8080/login?user=admin&callback=onBack得到的js文件，就是我们之前定义的onBack数,并且它的参数就是我们需要的json数据，这样我们就跨域获得了我们需要的数据。

document.domain+iframe

1.原理：浏览器的同源政策其中限制了浏览器中不同域的框架之间是不能进行js的交互操作的。有一点需要说明，不同的框架之间（父子或同辈），是能够获取到彼此的window对象的，但是却不能使用获取到的window对象的属性和方法(html5中的postMessage方法是一个例外，还有些浏览器比如ie6也可以使用top、parent等少数几个属性)，总之，你可以当做是只能获取到一个几乎无用的window对象。这个时候，document.domain就可以派上用场了，我们把需要跨域交互的两个页面的document.domain都设成相同的域名（一般用顶级域）。

2.注意：

这两个域名必须属于同一个基础域名!而且所用的协议，端口都要一致，否则无法利用document.domain进行跨域；

document.domain的设置是有限制的，我们只能把document.domain设置成自身或更高一级的父域，且顶级域必须相同；

只修改两个网页document.domain的方法只适用于不同子域的框架间的交互；

在不为子域的网页下想要使用document.domain方法跨域需要将其先修改为父子域。

3.代码如下：

A窗口 http://domain.com/

B窗口 http://www.domain.com/

C窗口 http://js.domain.com/

子域：B/C两个页面相互访问

//B网页
<script>
  document.domain = 'domain.com';
  var user = 'admin';
</script>

//C网页
<script>
  document.domain = 'domain.com';
  //获取A窗口变量
  console.log(window.user);   //admin
</script>

父子域：A/B两个页面相互访问 

//A网页
<iframe id='iframe' src='http://www.domain.com/'></iframe>
<script>
  document.domain = 'domain.com';
  var user = 'admin';
</script>

//B网页
<script>
  document.domain = 'domain.com';
  //获取A窗口变量
  console.log(window.parent.user);  //admin
</script>

window.name+iframe

1.原理：window对象有个name属性，该属性有个特征：即在一个窗口(window)的生命周期内,窗口载入的所有的页面（甚至不同域下）都是共享一个window.name的，每个页面对window.name都有读写的权限，window.name是持久存在一个窗口载入过的所有页面中的，并不会因新页面的载入而进行重置。并且大小可达2M。

2.注意：

跨域页面里设置window.name = ‘要传输的值’

在要获取数据的页面使用一个隐藏的iframe来充当一个中间角色

iframe的src设置为要跨域页面

必须把这个隐藏的iframe的src设置成跟所在页面同一个域才行

3.代码如下：

b向a传参

location.hash + iframe

1.原理： a欲与b跨域相互通信，通过中间页c来实现。 三个页面，不同域之间利用iframe的location.hash传值，相同域之间直接js访问来通信。

2.代码如下：

domain1.com域：a.html->domain2.com域：b.html->domain1.com域：c.html

//a.html
<script>
  var iframe = document.createElement('iframe');
   iframe.src = 'http://www.domain2.com/b.html'+'?#user=yi.du';
   ifram.style = 'display:none';
   document.body.appendChild(iframe);
   function onCallback(res){
        alert(res);
   }
</script>

//b.html
<script>
  var iframe = document.createElement('iframe');
   iframe.src = 'http://www.domain1.com/c.html';
   ifram.style = 'display:none';
   document.body.appendChild(iframe);
   function onhashchange(){
      iframe.src = iframe.src + location.hash;
   };
</script>

//c.html
<script>
  window.onhashchange = function() {
  window.parent.parent.onCallback('hello:' + location.hash.replace(' #user=', ''));
  }
</script>

3.总结：简单来说，就是url字符串拼接的方法。

window.location.hash

postMessage

1.原理：postMessage是HTML5 XMLHttpRequest Level 2中的API，且是为数不多可以跨域操作的window属性之一，它可用于解决以下方面的问题：

a.） 页面和其打开的新窗口的数据传递

b.） 多窗口之间消息传递

c.） 页面与嵌套的iframe消息传递

d.） 上面三个场景的跨域数据传递

2.用法：

postMessage(data,origin)方法接受两个参数：

data： html5规范支持任意基本类型或可复制的对象，但部分浏览器只支持字符串，所以传参时最好用JSON.stringify()序列化。

origin： 协议+主机+端口号，也可以设置为”*”，表示可以传递给任意窗口，如果要指定和当前窗口同源的话设置为”/“。
